Installation, mise en service

Pour faire fonctionner ce truc, il vous faudra les éléments suivants :

  • PyCharm, recommandé (pour Maccaud!)
  • Serveur MySQL
    • WampServer
    • Laragon
    • etc...
  • navigateur web
  • GitHub sera utilisé
  • Langage Python à installer
Il ne vous que peu d'éléments, mais il est important de les mettre en place correctement.
PyCharm est recommandé pour l'ouverture du projet afin de lancer le service qui fera office de "serveur" afin de gérer les actions disponible sur le site.
Prière également, lors de l'installation, de cocher les cases ASSOCIATIONS, ADD PATH, etc. Sans elle, PyCharm sera... bof.
Vous trouverez également à l'intérieur du fichier un document nommé requirements.txt vous indiquant les versions utilisées pour le bon fonctionnement du projet.
Pour garantir d'être sur les bonnes versions, nous recommandons PyCharm qui, pour sûr, vous mettra les bonnes versions.
L'usage du serveur MySQL est obligatoire afin d'utiliser la base de donnée. Lequel choisir est purement personnel et ne devrait pas impacter l'usage du programme.
Le Python est également un élément crucial, voici l'url Le navigateur est libre de votre choix. Une fois ces différents programmes installés, préparez-vous, ça va chauffer.

Mise en route

Vous voilà prêts à faire feu... Ca tombe bien, le grill vient de s'éteindre.

Mise en service bdd

Première étape, crucial, lancer votre serveur de base de donnée.
La page localhost devrait être fonctionnelle, c'est un bon indicateur chez la plupart des serveurs.
Il vous faut désormais remplir la base de donnée, à l'aide du dump fourni dans le dossier (APP_FILMS_164\database)

Mise en service PyCharm

Base prête, attaquons le code.
Commencer par ouvrir PyCharm et ouvrir le projet.
Créer un nouvel environnement pour le projet, vous devez avoir le 'venv' qui se crée. Mettre à jour le fichier '.env' situé à la racine du projet.
A partir de là, faites un backup de la base de donnée à utiliser. La stocker quelque part de safe.
Depuis PyCharm, ouvrir le dossier 'Database' et faire les différents tests.
Si quelque chose ne fonctionne pas bien, Voir la FAQ.
Si tout roule, lancer le 'run_mon_app.py'. Vous obtiendrez une URL et un port qui vont amèneront sur votre nouveau site!
